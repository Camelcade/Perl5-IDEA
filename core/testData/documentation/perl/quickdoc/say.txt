<dt style="padding-bottom:4px;font-weight:bold;">say FILEHANDLE</dt>
<dt style="padding-bottom:4px;font-weight:bold;">say LIST</dt>
<dt style="padding-bottom:4px;font-weight:bold;">say</dt><dd style="padding-top:6px;">
<p style="padding-bottom: 10px;">Just like <a href="psi_element://pod%3A%3Aperlfunc%2Fprint+FILEHANDLE+LIST">C<print></a>, but implicitly appends a
newline.  <code>say LIST</code> is simply an abbreviation for
<code>{ local $\ = &quot;\n&quot;; print LIST }</code>.  To use FILEHANDLE without a LIST to
print the contents of <a href="psi_element://perlvar%2F%24_">C<$_></a> to it, you must use a bareword
filehandle like <code>FH</code>, not an indirect one like <code>$fh</code>.</p>
<p style="padding-bottom: 10px;"><a href="psi_element://pod%3A%3Aperlfunc%2Fsay+FILEHANDLE+LIST">C<say></a> is available only if the
<a href="psi_element://feature%2FThe+%27say%27+feature" style="color:red">C<"say"> feature</a> is enabled or if it is
prefixed with <code>CORE::</code>.  The
<a href="psi_element://feature%2FThe+%27say%27+feature" style="color:red">C<"say"> feature</a> is enabled automatically
with a <code>use v5.10</code> (or higher) declaration in the current scope.</p></dd></dl>
