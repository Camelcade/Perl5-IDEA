<dt style="padding-bottom:4px;font-weight:bold;">int</dt><dd style="padding-top:6px;">
<p style="padding-bottom: 10px;">Returns the integer portion of EXPR.  If EXPR is omitted, uses
<a href="psi_element://perlvar%2F%24_">C<$_></a>.
You should not use this function for rounding: one because it truncates
towards <code>0</code>, and two because machine representations of floating-point
numbers can sometimes produce counterintuitive results.  For example,
<code>int(-6.725/0.025)</code> produces -268 rather than the correct -269; that's
because it's really more like -268.99999999999994315658 instead.  Usually,
the <a href="psi_element://pod%3A%3Aperlfunc%2Fsprintf+FORMAT%2C+LIST">C<sprintf></a>,
<a href="psi_element://pod%3A%3Aperlfunc%2Fprintf+FILEHANDLE+FORMAT%2C+LIST">C<printf></a>, or the
<a href="psi_element://POSIX%2FC%3Cfloor%3E" style="color:red">C<POSIX::floor></a> and <a href="psi_element://POSIX%2FC%3Cceil%3E" style="color:red">C<POSIX::ceil></a>
functions will serve you better than will <a href="psi_element://pod%3A%3Aperlfunc%2Fint+EXPR">C<int></a>.</p></dd></dl>
