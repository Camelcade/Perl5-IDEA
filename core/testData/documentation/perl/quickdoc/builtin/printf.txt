<dt style="padding-bottom:4px;font-weight:bold;">printf FILEHANDLE</dt>
<dt style="padding-bottom:4px;font-weight:bold;">printf FORMAT, LIST</dt>
<dt style="padding-bottom:4px;font-weight:bold;">printf</dt><dd style="padding-top:6px;">
<p style="padding-bottom: 10px;">Equivalent to <code>print FILEHANDLE sprintf(FORMAT, LIST)</code>, except that
<a href="psi_element://perlvar%2F%24%5C">C<$\></a> (the output record separator) is not appended.  The
FORMAT and the LIST are actually parsed as a single list.  The first
argument of the list will be interpreted as the
<a href="psi_element://pod%3A%3Aperlfunc%2Fprintf+FILEHANDLE+FORMAT%2C+LIST">C<printf></a> format.  This means that
<code>printf(@_)</code> will use <code>$_[0]</code> as the format.  See
<a href="psi_element://pod%3A%3Aperlfunc%2Fsprintf+FORMAT%2C+LIST">sprintf</a> for an explanation of the format
argument.  If <code>use locale</code> (including <code>use locale ':not_characters'</code>)
is in effect and <a href="psi_element://POSIX%2FC%3Csetlocale%3E" style="color:red">C<POSIX::setlocale></a> has been
called, the character used for the decimal separator in formatted
floating-point numbers is affected by the <code>LC_NUMERIC</code> locale setting.
See <a href="psi_element://perllocale">perllocale</a> and <a href="psi_element://POSIX" style="color:red">POSIX</a>.</p>
<p style="padding-bottom: 10px;">For historical reasons, if you omit the list, <a href="psi_element://perlvar%2F%24_">C<$_></a> is
used as the format;
to use FILEHANDLE without a list, you must use a bareword filehandle like
<code>FH</code>, not an indirect one like <code>$fh</code>.  However, this will rarely do what
you want; if <a href="psi_element://perlvar%2F%24_">C<$_></a> contains formatting codes, they will be
replaced with the empty string and a warning will be emitted if
<a href="psi_element://warnings" style="color:red">warnings</a> are enabled.  Just use <a href="psi_element://pod%3A%3Aperlfunc%2Fprint+FILEHANDLE+LIST">C<print></a> if
you want to print the contents of <a href="psi_element://perlvar%2F%24_">C<$_></a>.</p>
<p style="padding-bottom: 10px;">Don't fall into the trap of using a
<a href="psi_element://pod%3A%3Aperlfunc%2Fprintf+FILEHANDLE+FORMAT%2C+LIST">C<printf></a> when a simple
<a href="psi_element://pod%3A%3Aperlfunc%2Fprint+FILEHANDLE+LIST">C<print></a> would do.  The
<a href="psi_element://pod%3A%3Aperlfunc%2Fprint+FILEHANDLE+LIST">C<print></a> is more efficient and less error
prone.</p></dd></dl>
