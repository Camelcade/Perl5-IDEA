<dt style="padding-bottom:4px;font-weight:bold;">tell</dt><dd style="padding-top:6px;">
<p style="padding-bottom: 10px;">Returns the current position <i>in bytes</i> for FILEHANDLE, or -1 on
error.  FILEHANDLE may be an expression whose value gives the name of
the actual filehandle.  If FILEHANDLE is omitted, assumes the file
last read.</p>
<p style="padding-bottom: 10px;">Note the <i>in bytes</i>: even if the filehandle has been set to
operate on characters (for example by using the <code>:encoding(utf8)</code> open
layer), <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE">C<tell></a> will return byte offsets, not
character offsets (because that would render
<a href="psi_element://pod%3A%3Aperlfunc%2Fseek+FILEHANDLE%2CPOSITION%2CWHENCE">C<seek></a> and
<a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE">C<tell></a> rather slow).</p>
<p style="padding-bottom: 10px;">The return value of <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE">C<tell></a> for the standard streams
like the STDIN depends on the operating system: it may return -1 or
something else.  <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE">C<tell></a> on pipes, fifos, and
sockets usually returns -1.</p>
<p style="padding-bottom: 10px;">There is no <code>systell</code> function.  Use <code>sysseek($fh, 0, 1)</code> for that.</p>
<p style="padding-bottom: 10px;">Do not use <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE">C<tell></a> (or other buffered I/O
operations) on a filehandle that has been manipulated by
<a href="psi_element://pod%3A%3Aperlfunc%2Fsysread+FILEHANDLE%2CSCALAR%2CLENGTH%2COFFSET">C<sysread></a>,
<a href="psi_element://pod%3A%3Aperlfunc%2Fsyswrite+FILEHANDLE%2CSCALAR%2CLENGTH%2COFFSET">C<syswrite></a>, or
<a href="psi_element://pod%3A%3Aperlfunc%2Fsysseek+FILEHANDLE%2CPOSITION%2CWHENCE">C<sysseek></a>.  Those functions
ignore the buffering, while <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE">C<tell></a> does not.</p></dd></dl>
