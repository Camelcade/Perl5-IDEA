<dt style="padding-bottom:4px;font-weight:bold;">tell</dt><dd style="padding-top:6px;">
<p style="padding-bottom: 10px;">Returns the current position <i>in bytes</i> for FILEHANDLE, or -1 on
error.  FILEHANDLE may be an expression whose value gives the name of
the actual filehandle.  If FILEHANDLE is omitted, assumes the file
last read.</p>
<p style="padding-bottom: 10px;">Note the emphasis on bytes: even if the filehandle has been set to operate
on characters (for example using the <code>:encoding(UTF-8)</code> I/O layer), the
<a href="psi_element://pod%3A%3Aperlfunc%2Fseek+FILEHANDLE%2CPOSITION%2CWHENCE"><code>seek</code></a>,
<a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE"><code>tell</code></a>, and
<a href="psi_element://pod%3A%3Aperlfunc%2Fsysseek+FILEHANDLE%2CPOSITION%2CWHENCE"><code>sysseek</code></a>
family of functions use byte offsets, not character offsets,
because seeking to a character offset would be very slow in a UTF-8 file.</p>
<p style="padding-bottom: 10px;">The return value of <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE"><code>tell</code></a> for the standard streams
like the STDIN depends on the operating system: it may return -1 or
something else.  <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE"><code>tell</code></a> on pipes, fifos, and
sockets usually returns -1.</p>
<p style="padding-bottom: 10px;">There is no <code>systell</code> function.  Use
<a href="psi_element://pod%3A%3Aperlfunc%2Fsysseek+FILEHANDLE%2CPOSITION%2CWHENCE"><code>sysseek($fh, 0, 1)</code></a> for that.</p>
<p style="padding-bottom: 10px;">Do not use <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE"><code>tell</code></a> (or other buffered I/O
operations) on a filehandle that has been manipulated by
<a href="psi_element://pod%3A%3Aperlfunc%2Fsysread+FILEHANDLE%2CSCALAR%2CLENGTH%2COFFSET"><code>sysread</code></a>,
<a href="psi_element://pod%3A%3Aperlfunc%2Fsyswrite+FILEHANDLE%2CSCALAR%2CLENGTH%2COFFSET"><code>syswrite</code></a>, or
<a href="psi_element://pod%3A%3Aperlfunc%2Fsysseek+FILEHANDLE%2CPOSITION%2CWHENCE"><code>sysseek</code></a>.  Those functions
ignore the buffering, while <a href="psi_element://pod%3A%3Aperlfunc%2Ftell+FILEHANDLE"><code>tell</code></a> does not.</p></dd></dl>
