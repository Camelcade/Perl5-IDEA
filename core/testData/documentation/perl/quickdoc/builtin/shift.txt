<dt style="padding-bottom:4px;font-weight:bold;">shift</dt><dd style="padding-top:6px;">
<p style="padding-bottom: 10px;">Shifts the first value of the array off and returns it, shortening the
array by 1 and moving everything down.  If there are no elements in the
array, returns the undefined value.  If ARRAY is omitted, shifts the
<a href="psi_element://perlvar%2F%40_">C<@_></a> array within the lexical scope of subroutines and
formats, and the <a href="psi_element://perlvar%2F%40ARGV">C<@ARGV></a> array outside a subroutine
and also within the lexical scopes
established by the <code>eval STRING</code>, <code>BEGIN {}</code>, <code>INIT {}</code>, <code>CHECK {}</code>,
<code>UNITCHECK {}</code>, and <code>END {}</code> constructs.</p>
<p style="padding-bottom: 10px;">Starting with Perl 5.14, an experimental feature allowed
<a href="psi_element://pod%3A%3Aperlfunc%2Fshift+ARRAY">C<shift></a> to take a
scalar expression. This experiment has been deemed unsuccessful, and was
removed as of Perl 5.24.</p>
<p style="padding-bottom: 10px;">See also <a href="psi_element://pod%3A%3Aperlfunc%2Funshift+ARRAY%2CLIST">C<unshift></a>, <a href="psi_element://pod%3A%3Aperlfunc%2Fpush+ARRAY%2CLIST">C<push></a>,
and <a href="psi_element://pod%3A%3Aperlfunc%2Fpop+ARRAY">C<pop></a>.  <a href="psi_element://pod%3A%3Aperlfunc%2Fshift+ARRAY">C<shift></a> and
<a href="psi_element://pod%3A%3Aperlfunc%2Funshift+ARRAY%2CLIST">C<unshift></a> do the same thing to the left end of
an array that <a href="psi_element://pod%3A%3Aperlfunc%2Fpop+ARRAY">C<pop></a> and <a href="psi_element://pod%3A%3Aperlfunc%2Fpush+ARRAY%2CLIST">C<push></a> do to
the right end.</p></dd></dl>
